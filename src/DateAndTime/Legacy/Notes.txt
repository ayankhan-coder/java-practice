1. set() – User chooses an exact value

Use case: Booking a flight.

Suppose a user selects:

Year: 2027
Month: March
Day: 15

The application simply sets those values.

Calendar cal = Calendar.getInstance();

cal.set(Calendar.YEAR, 2027);
cal.set(Calendar.MONTH, Calendar.MARCH);
cal.set(Calendar.DATE, 15);
Why set()?

Because the user is not adding days or months. They are choosing an exact date.

Other examples:

Setting an alarm for 7:00 AM.
Setting a meeting to 2 PM.
Setting a birthday.
----------------------------
2. add() – Calculate a future or past date

This is the most commonly used method in legacy applications.

Example 1: Subscription expires after 30 days

Suppose someone buys Netflix today.

Purchase Date:
July 11

The application needs the expiry date.

Calendar expiry = Calendar.getInstance();

expiry.add(Calendar.DATE, 30);

Result:

August 10

If today were December 20,

December 20 + 30 days

add() automatically gives

January 19 (next year)

It correctly changes both the month and year.
-----------------------------------------------
3. roll() – Change only one field (rarely used)

This method is rare in business applications.

It is mainly useful for calendar or date picker UIs where changing one field should not affect the others.

Example: Calendar Widget

Imagine this date picker:

Month: December
Day: 30
Year: 2025

The user presses the up arrow on the day field.

The UI changes:

30
↓

31
↓

1
↓

2

But the month should still display:

December

Using

cal.roll(Calendar.DATE, 1);

the day wraps within December instead of advancing to January.

This lets the user edit the day independently before confirming the full date.

Example: Digital Clock UI

Suppose a user is editing minutes.

Current time:

10 : 59

They press the up button.

The UI becomes

10 : 00

not

11 : 00

because only the minutes should wrap while the hour stays the same until the user finishes editing.

This is a typical use for roll().

Why isn't roll() used much?

Imagine an online shopping app.

Delivery date:

December 31

Adding one day should result in

January 1

Using roll() would produce

December 1

which is incorrect for real-world date calculations.

That's why business applications almost always use add() for date arithmetic.

Which one is used the most?
Method	Real-world example	Usage frequency
set()	User selects a date/time, alarm, birthday	⭐⭐⭐⭐ Very common
add()	Expiry dates, subscriptions, hotel stays, loans, reminders	⭐⭐⭐⭐⭐ Extremely common (legacy code)
roll()	Date picker or clock UI field editing	⭐ Rare

TIME ZONE NOTES

# Java TimeZone Notes (Old Date & Time API)

## 1. What is TimeZone?

* `TimeZone` is a class in `java.util`.
* It **does NOT store the current time**.
* It stores the **time zone rules** of a particular region (India, Japan, Singapore, etc.).

Example:

```java
TimeZone tz = TimeZone.getTimeZone("Asia/Tokyo");
```

This means:

> "Give me Tokyo's time zone rules."

---

## 2. Why do we need TimeZone?

Different countries have different local times.

Example:

* India → UTC +5:30
* Singapore → UTC +8:00
* Tokyo → UTC +9:00

Applications like Google Meet, Flight Booking, Amazon, Banking, etc. must display time according to the user's country.

---

## 3. Important Methods

### Get Computer's Default TimeZone

```java
TimeZone tz = TimeZone.getDefault();
```

Returns the operating system's configured time zone.

---

### Get Any Specific TimeZone

```java
TimeZone tz = TimeZone.getTimeZone("Asia/Singapore");
```

Returns Singapore's TimeZone object.

---

### Get TimeZone ID

```java
tz.getID();
```

Example Output:

```
Asia/Singapore
```

---

### Get Display Name

```java
tz.getDisplayName();
```

Example Output:

```
Singapore Standard Time
```

---

### Get Raw Offset

```java
tz.getRawOffset();
```

Returns the difference from UTC in milliseconds.

Example:

```
28800000
```

(8 hours)

---

## 4. Using TimeZone with Calendar

```java
TimeZone tz = TimeZone.getTimeZone("Asia/Tokyo");

Calendar cal = Calendar.getInstance(tz);

System.out.println(cal.get(Calendar.HOUR_OF_DAY));
```

Meaning:

1. Get Tokyo's TimeZone.
2. Create a Calendar using Tokyo's time rules.
3. Get Tokyo's current hour.

---

## 5. Two Ways of Writing

### Recommended (Easy to Understand)

```java
TimeZone tz = TimeZone.getTimeZone("Asia/Singapore");

Calendar cal = Calendar.getInstance(tz);
```

### Short Form

```java
Calendar cal = Calendar.getInstance(
        TimeZone.getTimeZone("Asia/Singapore"));
```

Both are exactly the same.

---

## 6. Important Confusion

This is WRONG:

```java
TimeZone tz = TimeZone.getTimeZone("Asia/Tokyo");

tz.getTime();   // ❌ Doesn't exist
```

Reason:

TimeZone is NOT a clock.

It only stores time zone information.

Calendar uses those rules to calculate time.

---

## 7. Why did getTime() show IST?

```java
Calendar cal = Calendar.getInstance(
        TimeZone.getTimeZone("Asia/Singapore"));

System.out.println(cal.getTime());
```

Output may still show:

```
IST
```

Reason:

`getTime()` returns a `Date` object.

`Date.toString()` prints using the computer's default time zone.

This is one of the biggest problems of the old Java Date-Time API.

---

## 8. Real-World Usage

TimeZone is commonly used with:

* Calendar
* SimpleDateFormat
* Date Formatting
* Flight Booking Systems
* Banking Applications
* Video Meeting Applications
* International Websites

---

## 9. Interview Question

**Q:** Does TimeZone store current time?

**Answer:**

No.

It only stores the rules of a particular time zone.

Other classes like Calendar use those rules to calculate date and time.

---

## 10. Most Important Point

Remember this sentence:

> **TimeZone is a helper object. It is rarely used alone. It provides time zone rules that other classes (Calendar, SimpleDateFormat, etc.) use to calculate or display time correctly.**
